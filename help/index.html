IF ... THEN ...

The simplest way of programming bucket is with a IF x THEN y message. Bucket will then respond with Y whenever someone mentions X in a message.

IF trigger THEN I was triggered.
Okay, if someone says 'trigger' then I'll reply "I was triggered.".
Hey Bucket, were you triggered?
I was triggered.

IF ... CHOOSE ...

Bucket can also be programmed using the CHOOSE keyword. Messages of the form IF x CHOOSE y;z;a;b will cause bucket to respond to any mention of X with a random choice from Y Z A B. A semicolon is the delimiter for choices.

IF best pet CHOOSE A dog; A cat; A mouse
Okay, if someone says 'best pet' then I'll reply with one of "A dog", "A cat", "A mouse".
Hey Bucket, whats the best pet?
A cat
Hey Bucket, whats the best pet?
A dog
Hey Bucket, whats the best pet?
A mouse

Items

Bucket is able to hold up to 30 items. You can give an item to bucket by using the Give Bucket X syntax. Bucket will add X to its items list. If bucket is already holding 30 items, it will drop one.

Bucket can also be told to give out an item by using the Bucket give X a present Bucket will respond by randomly choosing an item it is holding and ""give"" that item to X.

Give bucket a biscuit
*Bucket is holding a biscuit*
Bucket give John a present
*Bucket gave John a biscuit*

Timers

Bucket can be programmed to do timed messages. Bucket will check every minute to see whether it has any timers to trigger.

To program a time use the form Bucket CRONSYNTAX msg where CRONSYNTAX is a cron timer string. More infomration on how to write a cron string can be found at https://crontab.guru/

Bucket 5 * * * * * Hello
Okay, I will say 'Hello' every 5 minutes.
Hello

Dialog trees

It is possible to write complex dialog trees for bucket. The inital trigger will cause bucket to move down the tree and will respond to all subsequent triggers if they occur. Bucket can only move down a dialog tree and any message that does not correspond to a programmed trigger will cause bucket to leave the dialog tree.

Dialog trees are difficult to write, below is a handy tool that will convert the visual diagram of the dialog tree into a programming message that can be sent directly to bucket.


Keywords

Bucket has several keywords that can be utilised in responses. All keywords start with a $.
KeywordResult
$USERThe first name of the user who triggered the response.
$RANDOMThe first name of a random user.
$RAND###A random integer between 0 and the value of ###. e.g. $RAND100
$FILE:path-to-fileThis will attempt to parse and post the file stored at the provided URL. This can be used to send images, videos and sounds.
$TIMEOUTThis will cause whoever triggered the response to be kicked out of the chat for 10 minutes. Bucket will then add that person back once the alotted time has passed.

Bucket also has preloaded dictionaries for random word and sentence generation. These keywords can be used directly $KEYWORD for a random word, or can be used with a specific prefix. To generate a random word with a specific prefix use $PREFIX_KEYWORD. e.g To generate a random noun begining with P, use $P_NOUN.

KeywordResult
$NOUNA random noun.
$NOUNSA random plural noun.
$VERBA random verb (infinitive).
$VERBSA random verb (third person singular).
$VERBEDA random verb (past).
$VERBINGA random verb (present).
$ADJECTIVEA random adjective.
$ADVERBA random adverb.
$GENREA random musical genre.

No More

Bucket can remove a trigger from its database using Bucket no more X.

Bucket no more X
Okay, I wont respond to "X" anymore.

Global quiet

Bucket can be set to quiet for a period of time using Bucket shut up ## where ## to a number in minutes. Bucket will not respond to any messages during this period.

Bucket shut up 10
Okay, I wont respond to any messages for 10 minutes.
